# crush - 用于代码分析的 AI 助手

`crush` 是由 Charm 开发的 AI 驱动工具，提供上下文感知的建议和代码解释，并与语言服务器协议 (LSP) 集成，提供流畅的开发体验。

## 激活时机
- 当用户想要交互式的 AI 辅助进行代码分析时。
- 当在项目内进行快速诊断或寻求智能建议时。
- 当运行单个、非交互式的 AI 提示词进行代码解释时。
- 当用户需要调试或以 YOLO 批准模式执行 AI 命令时。

## 核心原则与规范
- **API 密钥继承**: 从 `x openai` 和 `x gemini` 配置中继承 `OPENAI_API_KEY` 和 `GEMINI_API_KEY`。
- **上下文感知**: 最好在特定项目目录 (`--cwd`) 中使用，以利用本地代码上下文。
- **YOLO 模式**: 使用 `-y` 或 `--yolo` 自动接受权限，以进行高速交互。

## 补充场景
- **非交互式提示**: 使用 `run` 子命令进行快速查询。
- **调试**: 使用 `-d` 启用详细日志，以排除 AI 交互故障。

## 实战示例

### 交互式会话
```bash
# 为当前项目启动交互式 AI 会话
x crush
```

### 非交互式查询
```bash
# 请求快速的代码解释
x crush run "解释 Go 语言中 context 的用法"
```

### 指定工作目录并开启调试
```bash
# 在特定项目目录中运行 crush 并开启调试日志
x crush -d -c /path/to/project
```
