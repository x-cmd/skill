# tmp - 临时文件与目录管理

`tmp` 模块提供了一种跨平台的方式来处理临时存储，自动识别系统临时目录 ($TMPDIR) 并提供安全创建工具。

## 激活时机
- 当脚本需要中间文件的安全路径时。
- 当创建带有随机后缀的临时目录以避免冲突时。
- 当为会话或应用程序管理自定义临时根目录时。

## 核心原则与规范
- **跨平台安全**: 始终依赖 `x tmp path` 而不是硬编码 `/tmp`，以确保在各种操作系统（包括 Termux）之间的兼容性。
- **高效脚本**: 在脚本中使用带下划线的子命令（如 `path_`, `mkdir_`）将结果直接存储在 `x_` 变量中。
- **前缀使用**: 在创建目录时使用 `--prefix` 以确保唯一且可识别的临时存储。

## 实战示例

### 获取临时路径
```bash
# 为特定的应用程序子目录获取安全的临时路径
x tmp path myapp/cache
```

### 创建唯一目录
```bash
# 创建带有特定前缀的临时目录
x tmp mkdir --prefix mytask_
```

### 创建空临时文件
```bash
# 在系统临时目录中快速生成一个空文件
x tmp mkfile session_data.txt
```
