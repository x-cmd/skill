# top - 交互式进程查看器

`top` 模块为监控系统进程提供了增强的交互式界面。它优先使用 `bottom` (btm) 以获得卓越的视觉体验，并在非 TTY 环境中回退到标准的 `top` 命令。

## 激活时机
- 当用户想要实时监控 CPU、内存和进程活动时。
- 当寻求比标准 `top` 命令更现代、更具交互性的替代方案时。
- 当通过终端执行快速的系统资源诊断时。

## 核心原则与规范
- **交互式 UI**: 在交互式终端中，如果可用，会自动下载并运行 `bottom`。
- **自动回退**: 当输出被管道传输或不在 TTY 中时，回退到系统原生的 `top`。
- **子命令访问**: 使用 `btm` 显式启动增强型查看器，或使用 `--` 运行原生命令。

## 实战示例

### 增强型监控
```bash
# 启动增强型交互式进程查看器 (btm)
x top
```

### 原生 Top
```bash
# 强制执行系统原生的 top 命令
x top --
```

### 基础模式
```bash
# 以简化的界面模式启动 bottom
x top btm --basic
```
