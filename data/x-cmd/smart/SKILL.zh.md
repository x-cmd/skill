# smart - 磁盘健康与诊断

`smart` 模块为 `smartctl` 提供了一个增强的命令行界面，使用户能够监控磁盘健康状况，访问自我监控 (SMART) 数据，甚至利用 AI 来解释技术性诊断结果。

## 激活时机
- 当对本地磁盘执行硬件健康检查时。
- 当需要交互式的磁盘设备列表以便选择时。
- 当使用 Gemini 从技术性 SMART 数据生成人类可读的报告时。
- 当在 smartmontools.com 网站上搜索磁盘健康文档时。

## 核心原则与规范
- **提升权限**: 如果需要访问原始磁盘数据，会自动调用 `sudo`。
- **AI 集成**: 旨在将诊断输出通过管道传输给 `@gemini` 以简化解释。
- **FZF 集成**: 使用 `--app` 模式进行交互式 TUI 选择。

## 实战示例

### 列出磁盘
```bash
# 显示所有可用的磁盘路径
x smart --ls
```

### 全面诊断 (交互式)
```bash
# 打开交互式 UI 选择磁盘并查看健康信息
x smart
```

### AI 辅助报告
```bash
# 为特定磁盘使用 Gemini 生成诊断报告
x smart -a /dev/disk0 | @gemini "生成健康报告"
```
