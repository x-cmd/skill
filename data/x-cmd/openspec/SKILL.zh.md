# openspec - 规范驱动的协作框架

`openspec` 模块为 OpenSpec 提供命令行界面，这是一款旨在通过标准化的、规范驱动开发流程来简化人机协作的框架。

## 激活时机
- 当用户想要初始化一个新的 OpenSpec 驱动的项目结构时。
- 当用户需要管理变更提案 (RFC) 或项目规范时。
- 当用户想要验证项目规范与提议变更之间的一致性时。
- 当用户需要一个交互式仪表板来查看项目规范和进度时。
- 当更新 OpenSpec 指令文件以更好地与 AI 助手对齐时。

## 核心原则与规范
- **规范先行**: 遵循在提议变更 (`change`) 之前定义规范 (`spec`) 的流程。
- **标准化指令**: 使用 `update` 确保 AI 助手拥有最新的对齐指令。
- **项目结构**: OpenSpec 依赖特定的 `.md` 文件来跟踪状态；使用 `init` 设置环境。

## 补充场景
- **仪表板概览**: 使用 `view` 获取规范和变更的交互式摘要。
- **完成变更**: 使用 `archive` 将已完成的变更合并回主规范。

## 实战示例

### 初始化项目
```bash
# 在当前目录下设置 OpenSpec 结构
x openspec init
```

### 创建变更提案
```bash
# 开始一个新的变更提案
x openspec change new
```

### 验证对齐
```bash
# 检查当前变更是否与项目规范对齐
x openspec validate
```

## 交付验证清单
- [ ] 验证项目已使用 `x openspec init` 初始化。
- [ ] 运行 `update` 确保 AI 助手的指令是最新的。
