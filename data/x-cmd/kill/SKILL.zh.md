# kill - 增强型进程终止工具

`kill` 模块通过提供高级功能（如递归终止进程树或根据进程占用的网络端口进行针对性终止），扩展了系统终止进程的能力。

## 激活时机
- 当用户需要终止监听标准端口（80, 443）的所有进程时。
- 当需要可用信号的 TUI 视觉呈现时。
- 当执行递归进程终止时。
- 当为了自动化而检查 JSON 格式的信号列表时。

## 核心原则与规范
- **针对性终止**: 使用 `byport` 专门清理 Web 服务端口。
- **视觉信号列表**: 使用 `-l` 通过交互式 TUI 探索信号。
- **JSON 支持**: 将信号导出为 JSON 以便于编程访问。

## 实战示例

### 按端口杀死进程
```bash
# 终止监听 80 和 443 端口的所有进程
x kill byport
```

### 信号列表 (TUI)
```bash
# 交互式浏览可用的系统信号
x kill -l
```

### 信号列表 (JSON)
```bash
# 以 JSON 格式获取信号信息
x kill -l --json
```

## 交付验证清单
- [ ] 确认要针对的具体 PID 或端口。
- [ ] 如果使用手动信号，请验证信号类型（默认为 SIGTERM）。
