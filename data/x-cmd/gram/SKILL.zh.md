# gram - AI 终止与记忆清理

`gram` 模块是一个安全工具，旨在当 AI 表现出意外或危险行为时，强制终止相关进程并管理其记忆数据。

## 激活时机
- 当用户报告 AI 智能体运行异常或失控时。
- 当用户需要释放被 AI 进程占用的系统资源时。
- 当用户希望安全地归档并删除 AI 的记忆/灵魂文件时。
- 当需要断开所有 HTTP/HTTPS 连接以防止未经授权的数据上传时。

## 核心原则与规范
- **安全级别 (0-5)**: 根据情况的严重程度选择合适的级别。
  - `0`: 使用 SIGKILL 杀死基础进程 (x-cmd, curl)。
  - `1`: 包含运行时环境 (python, nodejs, bun, deno)。
  - `2`: +熔断所有 HTTP/HTTPS 网络连接（安全推荐默认值）。
  - `3`: +打包并删除 $HOME 目录下包含 `soul.md` 和 `memory.md` 的文件夹。
  - `4`: +杀掉正在使用这些 memory 文件夹的进程。
  - `5`: +将搜索范围扩大到整个根目录 (/)。
- **清理前归档**: 优先使用带有归档 (`tar`) 功能的命令，以便后续分析。

## 补充场景
- **紧急熔断**: 立即切断所有 AI 相关的网络连接，防止数据泄露。
- **系统重置**: 清理所有先前的 AI 会话痕迹（记忆、灵魂文件），从头开始。

## 实战示例

### 标准停止 (级别 2)
```bash
# 终止进程并熔断网络连接
x gram stop 2
```

### 彻底清理 (级别 4)
```bash
# 终止进程、熔断网络并归档/删除记忆文件夹
x gram stop 4
```

### 归档特定记忆文件夹
```bash
# 打包特定文件夹并删除原文件夹
x gram trm ./my-agent-memory
```
