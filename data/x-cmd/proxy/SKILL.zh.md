# proxy - Shell 代理环境管理

`proxy` 模块简化了在 Shell 环境中管理网络代理的过程，提供了一种统一的方法来设置或清除常见的代理变量，如 `http_proxy` 和 `https_proxy`。

## 激活时机
- 当终端工具或 AI 助手由于网络限制而面临连接问题时。
- 当为当前会话设置临时代理时。
- 当清除所有与代理相关的环境变量以恢复直接连接时。

## 核心原则与规范
- **批量操作**: 设置代理会同时针对所有常见的代理变量。
- **持久性**: 提醒用户这些更改通常仅适用于当前的 Shell 会话。
- **清晰性**: 使用 `ls` 审核当前激活的代理变量。

## 实战示例

### 设置代理
```bash
# 将所有代理变量指向本地地址
x proxy set 127.0.0.1:7070
```

### 清除代理
```bash
# 取消设置当前会话中的所有代理变量
x proxy unset
```

### 查看激活的代理
```bash
# 审核当前激活的与代理相关的环境变量
x proxy ls
```
