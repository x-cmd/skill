# humantime - 人性化时间转换

`humantime` 模块通过将原始秒数转换为可读格式（如 `1h 30m`）以及反向转换，简化了时间操作。它支持小数秒和自动类型检测。

## 激活时机
- 当向用户显示脚本执行时长时。
- 当根据人性化输入计算缓存过期时间时。
- 当检查一个时长是否在特定限制内时（使用 `in` 子命令）。

## 核心原则与规范
- **自动检测**: 输入整数（例如 `3600`）会转换为时间格式 (`1h`)，而输入时间字符串（例如 `5m`）会转换为秒数 (`300`)。
- **时间单位**: 支持 `d` (天)、`h` (小时)、`m` (分钟) 和 `s` (秒)。
- **高精度**: 能够准确处理小数秒（例如 `3661.5s = 1h1m1.5s`）。

## 实战示例

### 将秒数转换为可读格式
```bash
# 将一天的时间转换为可读格式
x humantime 86400
```

### 将可读格式转换为秒数
```bash
# 获取 2 小时 30 分钟的原始秒数
x humantime 2h30m
```

### 范围内检查
```bash
# 检查 240 秒是否在 5 分钟内（如果是，则返回 0）
x humantime in 5m 240
```

## 交付验证清单
- [ ] 确认输入是整数（秒）还是字符串（可读格式）。
- [ ] 验证是否需要高精度（小数秒）。
