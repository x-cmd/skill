# tee - 增强型输出重定向

`tee` 模块执行命令并将其输出同时写入标准输出和指定文件。与传统的管道 `tee` 不同，它保留了原始命令的退出代码和环境变量修改。

## 激活时机
- 为 CI/CD 流水线记录构建、测试或部署输出。
- 在需要对命令失败做出反应时，同时捕获 stdout 和 stderr。
- 当命令修改了需要维护的环境变量时。

## 核心原则与规范
- **退出码完整性**: 对于后续逻辑取决于记录命令成功与否的脚本至关重要。
- **命令分隔**: 使用 `--` 分隔符区分输出文件和要运行的命令。
- **追加模式**: 支持使用 `-a` 将内容追加到日志文件而不是覆盖。

## 实战示例

### 记录构建输出
```bash
# 捕获构建日志，同时确保脚本在失败时停止
x tee build.log -- make build
```

### 捕获组合输出
```bash
# 将 stdout 和 stderr 同时记录到文件
x tee /tmp/task.log -- eval 'npm run test 2>&1'
```

### 记录部署过程
```bash
# 部署并保留日志，允许稍后检查退出代码
x tee deploy.log -- ./deploy.sh
```
