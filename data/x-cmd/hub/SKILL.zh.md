# hub - 云端脚本管理与 AI 搜索

`hub` 模块提供了一个安全的空间，用于在 x-cmd 云中管理和共享脚本。它支持加密托管、跨设备访问以及针对 x-cmd 知识库的 AI 驱动搜索。

## 激活时机
- 当用户想要将本地脚本上传到云端以便共享或跨设备使用时。
- 当在 x-cmd 知识库中执行 AI 搜索以查询特定模块的用法时。
- 当在远程云环境中编译 Go、Gop 或 Zig 脚本时。
- 当管理 x-cmd hub 账户和文件权限时。

## 核心原则与规范
- **安全性**: 支持加密托管以保护脚本隐私。
- **知识检索**: 使用 `search` 或 `ai` 子命令来利用 AI 知识库。
- **访问控制**: 使用 `access` 在公开和私有文件状态之间进行切换。

## 实战示例

### 上传并共享
```bash
# 上传文件并将其设置为公开
x hub file put --public my_script.sh
```

### AI 搜索
```bash
# 搜索如何通过 x-cmd 在 GitHub 仓库中添加成员
x hub search "如何在 github 仓库中添加成员"
```

### 账户信息
```bash
# 检查当前的云端空间使用情况和账户详情
x hub info
```

## 交付验证清单
- [ ] 确保用户已通过 `x hub login` 登录。
- [ ] 确认文件应该是公开的还是私有的。
- [ ] 验证 AI 搜索查询是否足够具体。
