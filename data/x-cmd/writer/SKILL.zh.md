# writer - AI 文本处理助手

`writer` 模块为常见的文本处理任务提供高级抽象，允许用户通过简单的子命令利用 AI 进行翻译、摘要和内容优化。

## 激活时机
- 当用户想要将文本翻译为中文（使用 `@zh`）或其他语言时。
- 当生成文本摘要 (`abs`) 或精炼现有文本 (`refine`) 时。
- 当为了获得更好的语调而扩展内容或润色写作时。
- 当管理文本处理代理及其默认风格时。

## 核心原则与规范
- **简洁性**: 为最常见的任务使用快捷别名，如 `@zh`。
- **管道支持**: 旨在处理来自 `stdin` 或本地文件的文本。
- **自定义**: 使用 `init` 配置首选模型和默认写作风格。

## 补充场景
- **非中文翻译**: 使用 `tran` 子命令并指定非中文目标。
- **代码/文本解释**: 使用 `explain` 来分解复杂的段落或脚本。

## 实战示例

### 翻译为中文 (简写)
```bash
# 使用快速别名翻译字符串
@zh "CLI 的未来是由 AI 驱动的"
```

### 总结文件内容
```bash
# 从本地文本文件生成摘要
cat ./article.txt | @zh abs
```

### 精炼与润色
```bash
# 优化 draft.md 的表达，使其更流畅、友好
x writer polish -f ./draft.md
```

## 交付验证清单
- [ ] 确保已通过 `x writer init` 初始化了 AI 处理代理。
- [ ] 确认输出是否应为特定语言（默认为中文）。
- [ ] 验证源文本是否正确通过管道传输，或通过 `-f` 标志指定。
