# os - 操作系统信息

`os` 模块提供有关当前操作系统的详细信息，包括其名称、架构、发行详情、平均负载等。这对于编写跨平台脚本至关重要。

## 激活时机
- 当用户需要识别系统架构时（例如，x86_64, arm64）。
- 当检查环境是否为 Linux、macOS、WSL 或 Termux 时。
- 当监控系统负载或运行时间时。
- 当获取发行版信息以进行软件兼容性检查时。

## 核心原则与规范
- **高效脚本**: 在脚本中使用 `name_`、`arch_` 和 `uptime_` 等变量进行高性能数据检索。
- **缓存控制**: 以 `get` 开头的子命令（如 `getname`）会绕过缓存进行实时检测。
- **跨平台准备**: 利用 `os is <类型>` 在 Shell 脚本中实现健壮的条件逻辑。

## 实战示例

### 基本信息
```bash
# 显示操作系统名称和架构
x os name
x os arch
```

### 环境检查
```bash
# 测试是否在 WSL 中运行（返回 0 或 1）
x os is wsl
```

### 系统性能
```bash
# 显示运行时间和平均负载
x os uptime
x os loadavg
```
