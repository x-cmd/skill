# ccmd - 命令输出缓存

`ccmd` 模块将命令的 stdout 缓存指定的时间，从而防止不必要地重复执行缓慢或资源密集型进程。

## 激活时机
- 当执行重复的网络请求时（例如 `curl`）。
- 当运行缓慢的诊断工具或复杂的分布式文件系统搜索时。
- 当用户想要在一段时间内保持命令输出的“快照”时。

## 核心原则与规范
- **时长格式**: 支持时间单位 `s`、`m`、`h`、`d`、`w`（例如 `1h` 表示一小时）。默认为 `1d`。
- **命令分隔**: 在提供要执行的命令时，`--` 分隔符是强制性的。
- **缓存管理**: 使用 `invalidate` 强制刷新，或使用 `clear` 删除所有缓存数据。

## 实战示例

### 缓存 HTTP 请求
```bash
# 将 curl 命令的结果缓存 1 小时
x ccmd 1h -- curl https://api.example.com/data
```

### 缓存本地查询
```bash
# 将地理 IP 查询缓存 30 分钟
x ccmd 30m -- x ips geo 1.1.1.1
```

### 强制刷新
```bash
# 清除特定命令的缓存以确保下次运行是实时的
x ccmd invalidate curl
```

## 交付验证清单
- [ ] 确认所需的缓存时长。
- [ ] 确保 `--` 分隔符放置正确。
- [ ] 验证命令输出是否适合缓存（即不是对实时性极度敏感的数据）。
