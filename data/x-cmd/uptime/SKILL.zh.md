# uptime - 结构化系统运行时间与负载

`uptime` 模块提供系统运行时长和平均负载（1、5 和 15 分钟）的结构化 YAML 输出，通过 `cosmo` 后端支持包括 Windows 在内的各种平台。

## 激活时机
- 当用户想要检查系统已运行了多长时间时。
- 当监控系统繁忙状态（CPU/IO 负载趋势）时。
- 当脚本需要结构化 (YAML) 的运行时间数据进行监控或记录时。

## 核心原则与规范
- **多平台**: 根据操作系统自动使用合适的后端（原生、busybox 或 cosmo）。
- **结构化**: 默认输出为 YAML，便于人类阅读和机器解析。

## 实战示例

### 结构化运行时间
```bash
# 以 YAML 格式显示系统运行时长和平均负载
x uptime
```

### 原始输出
```bash
# 获取原始的、非结构化的 uptime 命令输出
x uptime --raw
```
