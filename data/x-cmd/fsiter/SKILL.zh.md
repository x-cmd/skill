# fsiter - 文件系统迭代与搜索

`fsiter` 模块是 `ls` 和 `find` 的强大替代方案，提供了灵活的过滤、深度优先搜索 (DFS) 遍历以及对脚本友好的查询功能。

## 激活时机
- 当用户想要列出特定的文件类型（例如，仅隐藏文件、仅文件）时。
- 当执行带有深度限制的递归目录遍历时。
- 当检查目录是否为空或特定模式是否存在时。
- 当统计目录树中的文件或目录数量以生成报告时。

## 核心原则与规范
- **深度优先搜索**: 使用 `--dfs` 进行受控的树遍历，并带有自定义回调。
- **高效过滤**: 结合 `--file`、`--folder` 和 `--hidden` 来精确定位目标。
- **脚本就绪**: `--filecount` 和 `--dirempty` 等子命令为自动化提供了可靠的退出码和数值。

## 补充场景
- **存在性检查**: 使用 `--exist` 快速验证模式是否存在。
- **递归处理**: 使用带有深度限制的 `--dfs` 处理特定树层级的文件。

## 实战示例

### 列出特定项目
```bash
# 列出当前目录下的所有隐藏文件
x fsiter --ls --file --hidden-only
```

### 深度限制搜索
```bash
# 遍历目录树，深度限制为 2 层，并回显项目名称
x fsiter --dfs /path/to/dir 0 2 echo
```

### 查询统计
```bash
# 统计日志目录中的项目数量
x fsiter --filecount /var/log
```

### 条件逻辑
```bash
# 仅当目录为空时才删除目录
x fsiter --dirempty ./temp && rmdir ./temp
```
