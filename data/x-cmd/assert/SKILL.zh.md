# assert - 健壮的测试断言

`assert` 模块是一个功能强大的工具，用于验证测试结果是否符合预期。它广泛用于自动化测试和 CI/CD 流水线，以确保代码的正确性。

## 激活时机
- 当编写需要验证命令成功或失败的 Shell 脚本时。
- 当对文件系统执行回归测试时（检查非空、存在等）。
- 当验证变量状态或检测全局变量泄漏时。
- 当检查 stdout/stderr 是否符合预期模式或内容时。

## 核心原则与规范
- **关注退出码**: 命令成功时返回 0，失败时返回非 0，如果与 `set -e` 结合使用，可以中断脚本执行。
- **支持否定**: 使用 `!` 进行命令的否定检查，或使用 `^` 进行类型/文件的否定检查。
- **批量验证**: 许多子命令支持一次性检查多个值或文件。

## 补充场景
- **变量快照**: 使用 `var save` 和 `var cmp` 检测函数中意外的全局变量修改。
- **输出匹配**: 使用 `stdout` 结合 heredoc 验证复杂的、多行的命令输出。

## 实战示例

### 命令与类型验证
```bash
# 验证命令成功以及多个整数值
x assert true [ 1 -eq 1 ]
x assert is-int 10 20 30
```

### 文件系统检查
```bash
# 确保特定文件存在且非空
x assert is-file /etc/hosts
x assert is-nonempty ./log.txt
```

### 检测变量泄漏
```bash
# 捕获状态，运行函数，并检查泄漏
x assert var save
my_function
x assert var cmp
```

## 交付验证清单
- [ ] 确保使用了正确的操作符（`is-file`、`is-int` 等）。
- [ ] 确认测试逻辑是否需要否定（`!` 或 `^`）。
- [ ] 验证多行预期内容是否与 stdin/heredoc 格式匹配。
