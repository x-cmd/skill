# mosh - 稳健的远程 Shell (UDP)

`mosh` 模块为 Mobile Shell 提供了一个增强型包装器，通过在系统缺失时利用 pixi 自动下载，确保了工具的可获得性。它是漫游和高延迟连接的理想选择。

## 激活时机
- 当通过不稳定或高延迟的网络（如蜂窝数据）连接到远程服务器时。
- 当用户需要一个能在网络更替（漫游）过程中保持持久的连接时。
- 当寻求具有本地回显和行编辑功能的交互式 SSH 替代品时。

## 核心原则与规范
- **UDP 要求**: 使用 UDP 端口（默认 60000-61000）而不是 TCP。
- **SSH 依赖**: 需要一个正常运行的 SSH 命令来建立初始握手。
- **跨平台**: 在受支持的平台（macOS/Linux）上自动通过 `pixi` 处理设置。

## 实战示例

### 基础连接
```bash
# 使用 Mosh 连接到远程服务器
x mosh user@remote-host.com
```

### 特定端口与 SSH 命令
```bash
# 使用自定义 SSH 端口和特定的 Mosh UDP 端口
x mosh -p 80 --ssh="ssh -p 2222" root@server.com
```

## 交付验证清单
- [ ] 确保远程服务器也安装了 `mosh-server`。
- [ ] 验证远程防火墙已开放 UDP 端口 (60000-61000)。
- [ ] 确认目标用户和主机名。
