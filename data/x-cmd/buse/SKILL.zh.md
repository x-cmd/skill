# buse - 自然语言浏览器自动化

`buse` 模块使用户能够通过 `browser-use` 框架，使用自然语言控制网络浏览器。它能自动执行复杂的浏览器操作，而无需手动编写代码。

## 激活时机
- 当用户希望 AI 导航网站或与网页元素交互时。
- 当执行自动化的浏览器任务（如文档搜索或视觉验证/截图）时。
- 当使用终端 UI 交互式地引导浏览器代理时。
- 当为数据提取或测试运行无头浏览器任务时。

## 核心原则与规范
- **安全与权限**: AI 生成的浏览器操作应受到监控；在处理敏感网站时需谨慎。
- **模型要求**: 需要通过相应的 x-cmd 模块配置 AI 模型 API 密钥（OpenAI, Gemini 等）。
- **上下文处理**: 支持自定义浏览器窗口、用户数据目录和 CDP 连接。

## 补充场景
- **视觉证据**: 在浏览器任务期间使用特定提示词自动截取屏幕截图。
- **MCP 模式**: 作为模型上下文协议服务器运行，以便与其他 AI 工具集成。

## 实战示例

### 交互式浏览器控制
```bash
# 启动用于浏览器控制的交互式终端 UI
x buse
```

### 直接任务执行
```bash
# 以非交互方式运行特定的浏览器任务
x buse -p "搜索 OpenAI 文档并截取首页屏幕截图"
```

### 无头模式
```bash
# 在后台运行任务，不显示可见窗口
x buse --headless -p "检查 ExampleStore.com 上某商品的库存状态"
```

## 交付验证清单
- [ ] 确保已通过 `x buse --install` 安装了 browser-use 和 Chromium。
- [ ] 确认至少设置了一个 AI 模型 API 密钥。
- [ ] 验证任务是否需要可见窗口，或者可以在无头模式下运行。
